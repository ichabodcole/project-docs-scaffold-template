Pattern Reference: Tables with x-for

Two patterns that trip up Alpine.js inside <table> elements. Scroll down to see working live demos.

Pattern 1 — x-for inside a table

Problem: Placing <template x-for> inside <tbody> causes an importNode TypeError. The browser's HTML parser strips <template> tags when it encounters them inside <tbody>, so Alpine gets a null template.content. 

<!-- ❌ WRONG — importNode TypeError -->
<table>
  <tbody>
    <template x-for="item in items">
      <tr> ... </tr>
    </template>
  </tbody>
</table>

Fix: Wrap the <tbody> inside the x-for template instead. A <table> can have multiple <tbody> elements — this is valid HTML. 

<!-- ✅ CORRECT — each iteration gets its own <tbody> -->
<table>
  <thead> ... </thead>
  <template x-for="item in items" :key="item.id">
    <tbody>
      <tr> ... </tr>
    </tbody>
  </template>
</table>

Pattern 2 — Iterating objects with x-for

Problem: Using Object.entries() with array destructuring in x-for causes "X is not defined" ReferenceErrors. Alpine evaluates the loop variable expression in a scope where destructured variable names aren't available. 

<!-- ❌ WRONG — "[key, val] in Object.entries(obj)" causes ReferenceError -->
<template x-for="[key, val] in Object.entries(permissions)">
  <div x-text="key"></div>
  <div x-text="val.label"></div>
</template>

Fix: Iterate Object.keys() and access values via bracket notation inside the template. 

<!-- ✅ CORRECT — iterate keys, access values via bracket notation -->
<template x-for="key in Object.keys(permissions)">
  <div x-text="key"></div>
  <div x-text="permissions[key].label"></div>
</template>
