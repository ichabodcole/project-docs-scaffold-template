# Migration: v2.3 → v2.4

## Summary

v2.4 adds the **Test Plan** document type — an optional stage between Plan and
Sessions that defines tiered verification scenarios for agent-implemented work.
It also adds an **External Dependencies** subsection to the Design Resolution
template.

**Why this changed:** Agent-implemented features were being declared "complete"
without structured verification, leading to manual testing loops. The test plan
closes this gap by giving implementing agents a concrete verification roadmap
with prioritized scenarios. The external dependencies addition ensures
third-party setup requirements (API keys, accounts, environment variables) are
surfaced early rather than discovered during testing.

## What's New

| File/Directory                             | Purpose                          |
| ------------------------------------------ | -------------------------------- |
| `projects/TEMPLATES/TEST-PLAN.template.md` | Template for test plan documents |

## What Changed

## What's Removed

Nothing is removed in this migration.

## Step-by-Step Migration

### Run the Script First

```bash
SKILL_DIR="path/to/plugins/project-docs/skills/update-project-docs"

# Preview what will change (safe, no modifications)
bash "${SKILL_DIR}/migrations/scripts/migrate-v2.3-to-v2.4.sh" \
  --scaffold-dir ".scaffold-tmp/docs" --dry-run

# Apply mechanical changes (add --replace-* flags if files haven't been customized)
bash "${SKILL_DIR}/migrations/scripts/migrate-v2.3-to-v2.4.sh" \
  --scaffold-dir ".scaffold-tmp/docs" \
  [--replace-design-resolution-template] \
  [--replace-projects-readme]
```

The script handles: copying `TEST-PLAN.template.md`, optionally replacing
`DESIGN-RESOLUTION.template.md` and `docs/projects/README.md`, updating the
documentation cycle string, bumping `docs_version` to `2.4.0`, and scaffold
cleanup. Without the `--replace-*` flags, those files are left for agent merging
(steps 3 and 4 below).

### [Agent] 1. Generate v2.4 scaffold locally

```bash
cookiecutter gh:ichabodcole/project-docs-scaffold-template -o .scaffold-tmp
```

Then run the script above (passing the scaffold path), and continue with the
remaining agent steps below.

### [Agent] 3. Update Design Resolution template

_(Skip if you used `--replace-design-resolution-template`)_

The design resolution template gains an "External Dependencies (Optional)"
subsection under Architectural Positioning. Manually add the subsection after
the Architectural Positioning bullets — see the scaffold copy for the exact
content.

### [Agent] 4. Update docs/projects/README.md

_(Skip if you used `--replace-projects-readme`)_

Merge these additions into your `docs/projects/README.md`:

- `test-plan.md` in the project folder structure example (marked optional)
- A "What Goes Where" entry for `test-plan.md`
- A "Test Plans" subsection with when-to/when-not-to guidance and tier
  descriptions
- `TEST-PLAN.template.md` in the Templates list

See the scaffold's `docs/projects/README.md` for the exact content.

### [Agent] 5. Update docs/README.md

**5a. Decision flowchart** — Add the test-plan decision diamond between "Add
plan.md to project" and "Working on it?" in the flowchart. Find this block:

```
                Add plan.md to project
                         ↓
                  Working on it?
```

Replace it with:

```
                Add plan.md to project
                         ↓
              ┌──────────────────────┐
              │ Need structured      │
              │ verification?        │
              └──────────────────────┘
                    ↓           ↓
                  YES           NO
                    ↓           │
         Add test-plan.md       │
         to project (optional)  │
                    ↓           │
                    └─────┬─────┘
                          ↓
                  Working on it?
```

**5b. Documentation cycle** — _(handled by script)_ The script updates
`→ plan → sessions` to `→ plan → [test-plan] → sessions`.

**5c. Quick Reference** — Update the Project entry to mention test plan:

```markdown
- **Project:** "Here's the full story of building X — proposal, design
  resolution (optional), plan, test plan (optional), sessions, artifacts in one
  place."
```

**5d. Special cases** — Add:

```markdown
- **Need structured verification for agent-implemented features?** → Test plan
  after planning
```

**5e. Update version** — _(handled by script)_ The script bumps `docs_version`
to `2.4.0`.

### 6. Clean up scaffold — _(handled by script)_
